# トレラン大会 傷病者搬送ルート判断支援ツール 要件定義書

## 1. プロジェクト概要

### 目的

トレイルランニング大会において、山中で傷病者が発生した際に、安全かつ迅速な下山ルートを選定するための意思決定支援ツール。事前に登録したコース・エスケープルート情報をもとに、傷病者の位置を入力するとリアルタイムで下山口候補とその比較情報（距離・高低差）を提示し、救護スタッフが最適なルートを判断できるようにする。

### 利用フロー

#### 起動時

アプリ起動直後にスタート画面が表示され、以下の2択を選ぶ：

- **新規作成**：メインコースのGPXファイルを読み込み、編集モードで起動
- **開く**：既存のZIPファイルを読み込み、編集モードで起動

#### 編集モード（準備段階・大会前）

GPXを読み込んだ状態から、必要な情報を順次追加していく：

1. トレイル／ロード区間の設定（区間境界・区間名）
1. ポイントの登録（エイド・下山口・ヘリポート・駐車場など）
1. エスケープルートのGPX追加・分岐点設定
1. 車両アクセスルートの追加（任意）
1. ZIPファイルとして保存（エクスポート）

#### 確認モード（大会当日）

編集モードから切り替え、または起動時にZIPを読み込むと確認モードで使用できる：

1. 地図上に全ルート・ポイントが表示された状態で待機
1. 下山口・ヘリポートのオン／オフを必要に応じて切り替え
1. 傷病者発生時：位置を地図タップ／座標入力で指定
1. 下山口候補が**リアルタイムで**一覧表示される（距離・高低差）
1. スタッフが候補を比較して搬送ルートを決定する

> 編集モード⇔確認モードはいつでも切り替え可能。確認モードでは編集操作（ポイント追加・ルート変更）は無効化される。

### 対象ユーザー

- 大会本部スタッフ（PC・タブレット使用）
- 現場救護スタッフ（スマートフォン使用）

### アクセス方法

- ログイン不要（URLを知っていれば誰でもアクセス可能）
- 大会ごとに専用URLを発行する運用を想定

## 2. ルート構造の設計

### ルートの考え方

メインコースは**トレイル区間とロード区間が交互に繰り返す**構造を持つ。下山口はトレイル区間ごとに管理し、ロード区間には下山口を設定しない。

```
       下山口S
            ↑ 戻る
〈トレイル〉──A────B──〈ロード〉──C────D──〈トレイル〉──E──→ 下山口F
                 ↓                              ↓
             エスケープ1                    エスケープ2
                 ↓                              ↓
              下山口X                         下山口Y
```

傷病者がロード区間にいる場合は、ロード区間に下山口が存在しないため、隣接するトレイル区間の下山口のみが候補となる。

### 候補ルートの計算方針

傷病者の位置が特定されると、以下のすべての候補を自動計算して提示する：

|候補パターン        |説明                         |
|--------------|---------------------------|
|メインコース進行方向    |傷病者位置からコースを進んで次の下山口へ       |
|メインコース引き返し    |傷病者位置からコースを戻って手前の下山口へ      |
|エスケープルート（進行方向）|傷病者位置より先の分岐からエスケープして下山口へ   |
|エスケープルート（引き返し）|傷病者位置より手前の分岐に戻ってエスケープして下山口へ|

各候補について、傷病者位置から分岐点までの距離＋エスケープルートの距離を合算して計算する。

### 分岐点の扱い

- エスケープルートはメインコース上の「分岐点座標」と紐づける
- 分岐点はGPXアップロード後に地図上で手動指定する（自動検出は補助的に使用）

## 3. 主要機能

### 3-1. スタート画面・モード管理機能

**スタート画面：**

- アプリ起動時に表示される
- 「新規作成」：メインコースGPXファイルを選択して読み込み、編集モードで起動
- 「開く」：ZIPファイルを選択して読み込み、編集モードで起動

**モード切替：**

- メイン画面のヘッダーまたはサイドパネルにモード切替ボタンを常時表示
- 編集モード→確認モード：確認ダイアログを表示して切り替え
- 確認モード→編集モード：いつでも切り替え可能
- 確認モード中は編集操作（ポイント追加・ルート変更・区間設定）をすべて無効化
- 現在のモードをヘッダーに常時表示する

### 3-2. ルートデータ管理機能

- GPXファイルのアップロードによるルート取り込み
- ルートごとの属性設定：
  - ルート名
  - 種別（`course`：メインコース ／ `escape`：エスケープルート）
  - 難易度・危険度（低／中／高）
  - 搬送手段の適性（歩行可／担架可／ヘリ対応）
- エスケープルートの場合：メインコース上の分岐点を地図上で指定

**車両アクセスルートの登録（参考表示用）：**

- 下山口・ヘリポート・駐車場に到着した後の車道ルートを地図上に表示するための機能
- ルート計算には使用しない（スタッフへの視覚的な案内が目的）
- 追加方法：
  - 地図上をクリックして頂点を打ちポリラインを描く（手動描画）
  - GPXファイルをアップロードして取り込む
- 地図上では点線など、コース・エスケープと区別できるスタイルで表示
- 種別：`road_access`として管理

**トレイル／ロード区間の設定（メインコースのみ）：**

- GPXアップロード後、地図上でルートを区間分けする
- 各区間に種別（`trail`／`road`）を設定する
- トレイル区間には任意で**区間名**をつけられる（例：「序盤トレイル」「山頂稜線」）
- ロード区間は名前不要（空欄可）
- 区間の境界点はGPXの座標上で指定する
- 区間情報はrace.jsonに保存される（GPXファイル自体は変更しない）

**各要素の名前ルール：**

|要素                    |名前|デフォルト  |備考             |
|----------------------|--|-------|---------------|
|トレイル区間                |任意|空欄     |ロード区間は名前不要     |
|下山口（Pointの`name`）     |必須|—      |登録時に入力         |
|エスケープルート（Routeの`name`）|任意|紐づく下山口名|未入力時は自動で下山口名を使用|

区間設定のイメージ：

```
GPX座標列：  P0 ─ P1 ─ P2 ─ P3 ─ P4 ─ P5 ─ P6 ─ P7
区間種別：   |── trail ──|── road ──|──── trail ────|
```

### 3-3. ポイント管理機能

大会ごとに地図上のポイントを登録・管理する。

**ポイントの種別：**

|種別   |アイコン|ルート計算|説明                                                    |
|-----|----|-----|------------------------------------------------------|
|エイド  |🏕️   |対象外  |給水・補給ポイント（参考表示のみ）                                     |
|下山口  |🚩   |ゴール地点|徒歩搬送の終着点                                              |
|ヘリポート|🚁   |ゴール地点|下山口と同じ扱い。徒歩でそこに辿り着くことをゴールとしてルート計算の対象となる。実際のヘリ搬送手配は別途判断|
|駐車場  |🅿️   |対象外  |車両集結ポイント（参考表示のみ）                                      |
|カスタム |📍   |対象外  |任意の名前をつけて自由に登録                                        |

**ルート計算ゴール地点（下山口・ヘリポート）のオン／オフ機能：**

- 下山口・ヘリポートは候補に含めるかどうかをチェックボックスでオン／オフできる
- **準備フェーズ・運用フェーズ両方で切り替え可能**（当日の状況変化に対応）
- オフにしたポイントは地図上でグレー表示され、ルート計算の対象外となる
- サイドパネルのゴール地点一覧からまとめて切り替えできる
- オン／オフの状態はブラウザのローカルストレージに保持し、リロードしても維持される

**ポイントの追加方法：**

- 地図上をクリック／タップして位置を指定し、種別・名前を入力して登録
- 緯度・経度を手入力して登録

**ポイントの操作：**

- マーカーをクリックすると名前・種別・座標を表示
- 編集・削除が可能
- ポイント一覧をサイドパネルに表示

### 3-4. 地図表示機能

- 国土地理院地図をベースマップとして使用
- コース・エスケープルートをオーバーレイ表示（種別で色分け）
- 登録ポイントを種別ごとに異なるアイコンで表示
- 傷病者位置のマーカー表示
- ルート候補選択時：該当ルートをハイライト表示
- 車両アクセスルートを点線など参考表示用スタイルで表示（ルート計算には影響しない）

### 3-5. データ入出力機能（ZIPファイル）

大会データをZIPファイル単位でインポート・エクスポートできる。

**ZIPファイル構成：**

```
race_xxtrail_2026.zip
  ├── race.json              # 大会情報・ポイント・ルート定義・分岐点情報
  ├── course_main.gpx        # メインコースのGPX
  ├── escape_1.gpx           # エスケープルート1のGPX
  ├── escape_2.gpx           # エスケープルート2のGPX
  └── road_access_1.gpx      # 車両アクセスルートのGPX（任意）
```

**エクスポート：**

- 準備フェーズ完了後、現在の大会データをZIPで一括ダウンロード

**インポート：**

- ZIPファイルをアップロードするだけで大会データを復元
- 運用フェーズ開始時にこの操作を行う

### 3-6. 傷病者位置入力機能

以下の複数の方法で位置を指定可能：

- 地図上をタップ／クリックして指定
- GPS座標（緯度・経度）を手入力
- コース上のチェックポイント名から選択

### 3-7. ルート候補計算・表示機能（リアルタイム）

傷病者位置が指定されると即座に計算を実行し、すべての下山口候補を一覧表示する。

**表示項目：**

|項目       |説明                       |
|---------|-------------------------|
|下山口名     |ゴール地点の名前                 |
|ルート概要    |例：「メインコース進行」「エスケープ1経由・戻り」|
|距離       |傷病者位置から下山口までの合計距離（km）    |
|累積標高差（下り）|下りの累積標高差（m）              |
|累積標高差（上り）|引き返し時など上りが発生する場合（m）      |
|難易度・危険度  |低／中／高                    |
|搬送手段の適性  |歩行可／担架可／ヘリ対応             |

**候補に含める下山口の条件：**

- 下山口のオン／オフが「オン」のもののみ計算対象とする
- 傷病者がロード区間にいる場合、そのロード区間には下山口候補を生成しない（隣接トレイル区間の下山口のみ対象）

**UI：**

- カード形式で複数候補を並列表示
- 距離の短い順にデフォルト表示
- カードを選択すると地図上に該当ルートをハイライト

## 6. 技術要件

### 構成方針

- **フロントエンドのみで完結**（バックエンドサーバー不要）
- データの永続化はZIPファイルのエクスポート／インポートで代替
- ルート計算はすべてブラウザ上のJavaScriptで処理
- GitHub Pagesに静的サイトとしてデプロイ

### 技術スタック

|項目     |採用技術                      |備考            |
|-------|--------------------------|--------------|
|フレームワーク|React + TypeScript        |コンポーネント管理・型安全性|
|ビルドツール |Vite                      |高速ビルド・設定が少ない  |
|地図     |Leaflet.js + react-leaflet|国土地理院タイル対応    |
|GPXパース |gpxparser                 |シンプルで軽量       |
|ZIP処理  |JSZip                     |ブラウザでZIP読み書き可能|
|状態管理   |Zustand                   |軽量・シンプル       |
|スタイル   |Tailwind CSS              |レスポンシブ対応が速い   |
|ホスティング |GitHub Pages              |無料・GitHub連携   |

### プロジェクト構成

```
src/
  components/
    map/          # 地図関連コンポーネント
      MapView.tsx           # メイン地図
      RouteLayer.tsx        # コース・エスケープ表示
      PointMarkers.tsx      # ポイントマーカー表示
      CasualtyMarker.tsx    # 傷病者マーカー
    panels/       # サイドパネル
      EditPanel.tsx         # 編集モード用パネル
      OperationPanel.tsx    # 確認モード用パネル
      CandidateList.tsx     # ルート候補一覧
      ExitPointList.tsx     # 下山口オン／オフ一覧
    ui/           # 汎用UIコンポーネント
      StartScreen.tsx       # 起動時スタート画面
      ModeToggle.tsx        # モード切替ボタン
  hooks/
    useGpx.ts     # GPX読み込み・パース
    useRoute.ts   # ルート候補計算ロジック
    useZip.ts     # ZIP読み書き
  store/
    raceStore.ts  # 大会データ（ルート・ポイント）
    modeStore.ts  # 編集／確認モードの状態
    casualtyStore.ts  # 傷病者位置・ルート候補結果
  types/
    race.ts       # Race・Route・Point・Segmentの型定義
    candidate.ts  # ルート候補計算結果の型定義
  utils/
    geo.ts        # 距離・標高差計算（ haversine公式など）
    gpxParser.ts  # GPXパース・GeoJSON変換
    zipHandler.ts # ZIPエクスポート・インポート処理
```

### データフロー

```
ZIPインポート / GPX読み込み
        ↓
  raceStore（Zustand）にデータを格納
        ↓
  各コンポーネントがストアを購読
        ├── MapView：ルート・ポイントを地図に描画
        └── パネル：編集UIまたは確認UIを表示
        ↓
  編集操作 → raceStoreを更新 → 地図が再描画
  傷病者位置指定 → useRoute.tsで計算 → CandidateListに表示
        ↓
  ZIPエクスポート（raceStoreの内容をJSONとGPXに変換してZIP化）
```

### 地図データ

- 国土地理院タイル（無料・利用規約に準拠）
  - 標準地図：`https://cyberjapandata.gsi.go.jp/xyz/std/{z}/{x}/{y}.png`
  - 淡色地図：`https://cyberjapandata.gsi.go.jp/xyz/pale/{z}/{x}/{y}.png`

## 10. ルート計算ロジック設計

### 概要

傷病者の任意座標を受け取り、登録済みの全ルート・全ゴール地点（下山口・ヘリポート）の組み合わせに対してリアルタイムで距離・標高差を計算し、候補一覧を返す。

-----

### ステップ1：最近傍点（スナップ点）の特定

傷病者の任意座標Pから、各GPXルートの座標列に対して最近傍点を求める。

```
傷病者位置 P
     ↓ 垂線を下ろす
GPX座標列：P0 ─ P1 ─ P2 ─ [スナップ点N] ─ P3 ─ P4
```

**処理手順：**

1. GPXの各線分（Pi → Pi+1）に対して、Pから線分への最短距離を計算
1. 最短距離が最小の線分を特定し、その垂足を「スナップ点N」とする
1. スナップ点は `segmentIndex + ratio`（例：2.7 = P2とP3の間70%地点）として管理
1. **スナップ距離が100mを超えるルートは候補から除外する**

```typescript
type SnapPoint = {
  routeId: string
  segmentIndex: number  // 何番目の線分か
  ratio: number         // 線分上の位置（0.0〜1.0）
  lat: number
  lng: number
  ele: number           // 標高（線形補間）
  distanceFromCasualty: number  // 傷病者からの距離（m）
}
```

-----

### ステップ2：ゴール地点までの経路抽出

スナップ点Nから各ゴール地点までのGPX座標列のスライスを取り出す。

**メインコース（進行・引き返し）：**

```
進行方向：N → P3 → P4 → ... → ゴールA
戻り方向：N → P2 → P1 → ... → ゴールB（座標列を逆順に辿る）
```

**エスケープルート経由：**

```
（前方分岐）N →前進→ 分岐点J → エスケープGPX Q0 → Q1 → ... → ゴールX
（後方分岐）N →後退→ 分岐点J → エスケープGPX Q0 → Q1 → ... → ゴールX
```

乗り継ぎ点（分岐点J）はメインコースGPX上の座標として登録済みのため、
スナップ点Nから分岐点JまでのセグメントとエスケープGPXを連結して1本の経路として扱う。

-----

### ステップ3：距離・標高差の計算

抽出した座標列に対して順次計算する。

```typescript
// 距離：Haversine公式で隣接点間の球面距離を合算
function haversine(p1: LatLng, p2: LatLng): number // メートル単位

totalDistanceM = sum of haversine(Pi, Pi+1)

// 標高差：上り・下りを分けて累積
for each (Pi, Pi+1):
  diff = Pi+1.ele - Pi.ele
  if diff > 0: totalAscentM += diff
  if diff < 0: totalDescentM += Math.abs(diff)
```

GPXに標高データ（ele）が含まれない場合は標高差を「取得不可」として表示する。

-----

### ステップ4：候補一覧の生成

以下の組み合わせをすべて計算し、有効なものを候補リストとして返す。

|候補パターン     |経路                         |除外条件                 |
|-----------|---------------------------|---------------------|
|メイン進行      |N → 前進 → ゴール               |前方にゴールがない／スナップ距離100m超|
|メイン引き返し    |N → 後退 → ゴール               |後方にゴールがない／スナップ距離100m超|
|エスケープ（前方分岐）|N → 前進 → 分岐点J → エスケープ → ゴール|前方に分岐点がない            |
|エスケープ（後方分岐）|N → 後退 → 分岐点J → エスケープ → ゴール|後方に分岐点がない            |

**除外ルール：**

- ゴール地点のオン／オフが「オフ」のものは計算しない
- 傷病者がロード区間にいる場合、そのロード区間内のゴール地点は生成しない
- スナップ距離が100mを超えるルートは候補から除外

**ソート：**

- デフォルトは距離の短い順
- 候補カードに距離・累積下り標高差・累積上り標高差を表示

-----

### 計算結果の型定義

```typescript
type RouteCandidate = {
  id: string
  label: string           // 例：「エスケープ1経由（前方分岐）」
  exitPointName: string   // ゴール地点名
  exitPointType: 'exit' | 'helipad'
  segments: CandidateSegment[]
  totalDistanceM: number
  totalDescentM: number
  totalAscentM: number
  difficulty: 'low' | 'medium' | 'high'
  transportSuitability: ('walk' | 'stretcher' | 'helicopter')[]
}

type CandidateSegment = {
  routeId: string
  direction: 'forward' | 'backward'
  fromIndex: number   // スナップ点または分岐点
  toIndex: number     // 分岐点またはゴール
}
```

## 11. 状態管理設計（Zustand）

### ストア構成

ストアは3つに分割し、関心ごとに状態を管理する。

-----

### raceStore（大会データ）

ZIPまたはGPXから読み込んだデータをすべて保持する。

```typescript
type RaceStore = {
  // データ
  race: Race | null
  routes: Route[]
  points: Point[]

  // 読み込み・書き出し
  loadFromGpx: (file: File) => Promise<void>   // 新規作成時
  loadFromZip: (file: File) => Promise<void>   // 開く時
  exportToZip: () => Promise<void>             // 手動エクスポートのみ（自動保存なし）

  // ポイント操作
  addPoint: (point: Point) => void
  updatePoint: (id: string, patch: Partial<Point>) => void
  deletePoint: (id: string) => void

  // ルート操作
  addRoute: (route: Route) => void
  updateRoute: (id: string, patch: Partial<Route>) => void
  deleteRoute: (id: string) => void
  setSegments: (routeId: string, segments: Segment[]) => void
  setJunction: (routeId: string, junction: Junction) => void
}
```

-----

### modeStore（モード・操作状態）

編集／確認モードと、編集中のアクティブな操作ツールを管理する。
地図クリック時の動作はこのストアの状態で分岐する。

```typescript
type Mode = 'edit' | 'operation'

type EditTool =
  | 'none'
  | 'add_point'         // ポイント追加モード
  | 'set_segment'       // 区間境界設定モード
  | 'set_junction'      // 分岐点設定モード
  | 'draw_road_access'  // 車両アクセスルート描画モード

type ModeStore = {
  mode: Mode
  activeTool: EditTool

  setMode: (mode: Mode) => void
  setActiveTool: (tool: EditTool) => void
}
```

**地図クリック時の分岐ロジック：**

```
地図クリック
  ├── mode === 'edit'
  │     ├── activeTool === 'add_point'     → ポイント追加ダイアログを表示
  │     ├── activeTool === 'set_segment'   → 区間境界点を打つ
  │     ├── activeTool === 'set_junction'  → 分岐点を設定
  │     ├── activeTool === 'draw_road_access' → アクセスルートの頂点を追加
  │     └── activeTool === 'none'          → 何もしない
  └── mode === 'operation'
        → 傷病者位置を設定（casualtyStore.setPosition）
```

-----

### casualtyStore（傷病者・計算結果）

確認モード専用。傷病者位置とルート候補計算結果を保持する。
`setPosition` を呼ぶと同時にルート計算を起動する。

```typescript
type CasualtyStore = {
  position: LatLng | null
  candidates: RouteCandidate[]
  selectedCandidateId: string | null
  isCalculating: boolean

  setPosition: (pos: LatLng) => void    // セットと同時に計算を起動
  selectCandidate: (id: string) => void // 地図上のハイライトと連動
  clearCasualty: () => void
}
```

-----

### ZIPの読み書き方針

|タイミング|動作                                    |
|-----|--------------------------------------|
|新規作成 |GPX読み込み → raceStoreに格納 → 編集 → 手動エクスポート|
|開く   |ZIP読み込み → raceStoreに格納 → 編集 → 手動エクスポート|
|保存   |エクスポートボタンを押した時だけZIPを生成・ダウンロード         |
|自動保存 |なし（明示的な操作のみ）                          |

確認モード中の下山口オン／オフ状態は `localStorage` に保持し、ZIPには含めない。

## 13. 編集操作UX設計

### 区間境界の設定

```
1. サイドパネルの「区間境界を設定」ボタンを押す
         ↓
2. 地図上のメインコース（GPX線）をクリック
   → クリック位置に最も近いGPX座標点に境界マーカーが打たれる
         ↓
3. ポップアップが表示され、分割された区間ごとに設定する
   ・種別：トレイル ／ ロード
   ・区間名（トレイルのみ・任意）
         ↓
4. 「確定」で保存。境界マーカーは地図上に残り、後から移動・削除可能
```

### ポイントの追加

```
1. サイドパネルの「ポイントを追加」ボタンを押す（add_pointモード）
         ↓
2. 地図上の任意の場所をクリック
         ↓
3. ポップアップで設定する
   ・種別：エイド ／ 下山口 ／ ヘリポート ／ 駐車場 ／ カスタム
   ・名前（必須）
   ・備考（任意）
         ↓
4. 「追加」で保存。マーカーが地図上に表示される
```

### エスケープルートの追加

```
1. サイドパネルの「エスケープを追加」ボタンを押す
         ↓
2. GPXファイルを選択してアップロード
   → エスケープルートが地図上に表示される
         ↓
3. 「分岐点を設定」ボタンを押す（set_junctionモード）
         ↓
4. メインコース上の分岐点をクリック
   → 最近傍のGPX座標点にスナップして分岐点マーカーが打たれる
         ↓
5. ルート名・難易度・搬送手段の適性を入力して「確定」
```

### 車両アクセスルートの追加

```
【手動描画】
1. 「描画」ボタンを押す（draw_road_accessモード）
2. 地図上をクリックして頂点を追加していく
3. 「描画完了」で保存

【GPXアップロード】
1. 「GPX読込」ボタンを押してファイルを選択
2. 地図上に表示されたことを確認して「確定」
```

### モード切替

```
編集 → 確認：
  「確認モードへ」ボタンを押す
  → 確認ダイアログ「編集内容は保存しましたか？」
  → 「続ける」で確認モードへ切り替え
  → 編集操作がすべて無効化される

確認 → 編集：
  「←編集」ボタンを押す
  → 即座に切り替え（ダイアログなし）
  → 傷病者位置・計算結果はクリアされる
```
